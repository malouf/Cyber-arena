# Multiplayer Integration Plan (Convex)

## Concept
Arena Async is an asynchronous, simultaneous-turn combat game. Both players plan their moves in secret. Once both click "Commit Sequence", the server computes the outcome and sends the result back to both clients.

## The Secret Weapon: `engine.ts`
Thanks to our recent refactoring, `src/game/engine.ts` is a pure function. 
The multiplayer logic becomes incredibly elegant: **We just run `engine.ts` on the Convex server.**

## Phase 1: Database Schema (`convex/schema.ts`)
- `users`: Identity and rating.
- `matches`: 
  - `status`: 'matchmaking' | 'drafting' | 'playing' | 'completed'
  - `player1`: ID, Build, State (HP, PA, PM, Mana, Pos, Cooldowns)
  - `player2`: ID, Build, State
  - `turnNumber`: integer
  - `terrain`: Array of active traps/hazards
- `match_events`:
  - Stores the `CombatEvent[]` generated by the engine for each turn, allowing clients to replay the turn visually.

## Phase 2: The Networked Game Loop
1. **Matchmaking**: `api.matches.queue()` matches two players.
2. **Drafting**: Players submit `api.matches.submitDraft({ build })`. Once both submit, Convex initializes the match state and board positions.
3. **Planning Phase**: Players plan locally. No server calls are made during hex hovering or sequence building.
4. **Commit Phase**: Player calls `api.matches.submitTurn({ matchId, queue: actionQueue })`.
5. **Server Resolution (Convex Action/Mutation)**:
   - When Convex receives the second player's queue, it triggers the resolution.
   - It pulls the current game state from the DB.
   - It runs `const { events, nextTurnState } = resolveTurn(p1State, p2State, p1Queue, p2Queue)`.
   - It updates the DB with the new states and saves the `events`.
6. **Playback Phase**: The React clients, subscribed via `useSuspenseQuery`, detect the new turn. They read the `events` array from the DB and play out the animations exactly as they do in offline mode today.

## Security
Because the server executes the pure engine, client hacking is irrelevant. If a modified client sends an action queue trying to use an ability on cooldown, the `resolveTurn` logic running on Convex will simply ignore it or mark it as invalid.
